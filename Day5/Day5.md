# Comparing Prompt A vs Prompt B

## 1. Which roadmap feels more personalized?

**Prompt B, clearly.** Prompt A is a competent generic template — it assumes zero prior knowledge and treats "learn data science" as a one-size-fits-all problem. Prompt B reads like it was written *for one specific person*:

- It explicitly skips Python fundamentals because the person already has that skill
- Every daily task is sized to a real constraint (30–60 min/day) instead of an assumed "study session"
- It names the learning-style mix (video/project/reading) instead of guessing
- It ties the final outcome back to a stated career goal, not a generic "you'll know data science" ending

Prompt A could have been written for literally anyone. Prompt B could only have been written for *this* person, at *this* stage, with *this* amount of time.

## 2. Which roadmap would you actually follow?

**Prompt B** — and the reason is practical, not just aesthetic. Prompt A's Week 1 (full Python fundamentals) would waste the very first week for someone who already codes — that's the fastest way to disengage and quit by Day 5. Prompt A's daily tasks also don't respect a time budget; they read like "spend an evening on this," which doesn't fit 30–60 min/day. A roadmap only works if it's followable, and only Prompt B was actually built to fit the person's real schedule and skill level.

## 3. What role did context play in improving the result?

Context did three specific things here, not just "made it nicer":

- **Eliminated waste** — cut ~7 days of redundant fundamentals, redirecting that time to Pandas/stats/ML, which is what was actually needed
- **Made the plan physically executable** — pacing matched to 30–60 min/day instead of an assumed multi-hour block, which is the difference between a plan someone finishes and one they abandon
- **Anchored the plan to a goal** — every resource and project choice pointed at "Data Scientist," not a vague "learn some data science" outcome, so the portfolio project at the end actually serves the job search

The underlying prompt structure (weekly milestones → daily tasks → resources → projects → outcome) was identical in both. Context didn't change the *shape* of the output — it changed whether the *content* inside that shape was useful to the specific person asking.
